Les ListAdapter

  • Dcrivent les donnes afficher
    • CursorAdapter depuis des donnes Cursor

  • Sont responsable de construire les vues correspondant aux lments de la liste

Implmenter un adapteur

  • 3 mthodes surcharger :
    • getView qui retourne la vue fonction de l’index et potentiellement de la vue recycler
    • getCount qui retourne le nombre d'lments total de la liste

  • On appelle notifyDataSetChanged() quand les donnes ont chang pour forcer un recalcul de l’affichage

ScrapView

Pattern ViewHolder

  • Permet d'viter d’appeler plusieurs fois findViewById
  • On commence par dfinir un ViewHolderItem, avec les champs correspondants aux lments d’interface auxquels on voudra accder : 
static class ViewHolderItem {
    TextView textViewItem;
    TextView textViewItem2;
}

Pattern ViewHolder

  • Dans la mthode getView, on se sert alors de ce ViewHolder :
    • On ne fait les findViewById qu’une fois
    • On n’a plus qu' valoriser nos lments
  • On utilise les mthodes getTag et setTag pour mmoriser et rcuprer notre ViewHolder

Pattern ViewHolder

public View getView(int position, View convertView, ViewGroup parent) {
    ViewHolderItem viewHolder;
    if (convertView == null){
        LayoutInflater inflater = ((Activity) context).getLayoutInflater();
        convertView = inflater.inflate(R.layout.item, parent, false);
        viewHolder = new ViewHolderItem();
        viewHolder.textViewItem = (TextView) convertView.findViewById(R.id.textViewItem);
        viewHolder.textViewItem2 = (TextView) convertView.findViewById(R.id.textViewItem2);
        convertView.setTag(viewHolder);
    } else {
        viewHolder = (ViewHolderItem) convertView.getTag();
    }
    Item item = getItem(position);
    if (item != null) {
        viewHolder.textViewItem.setText(item.text);
        viewHolder.textViewItem2.setText(item.text2);
    }
    return convertView;
}
